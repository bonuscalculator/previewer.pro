Open Graph Protocol

Essential OG Meta Tags for Facebook

These are the Open Graph properties Facebook reads to build your link card. Each one controls a specific part of the preview.

og:title

Page Title

The main headline of the link card. Facebook truncates at ~88 characters on desktop. Keep your most important information front-loaded. Recommended: 60–90 characters.

og:description

Description

A brief summary shown below the title. Facebook typically shows 2–3 lines (~200–300 characters). Use it to complement the title and add a compelling reason to click.

og:image

Share Image

The dominant visual element of the card. Recommended: 1200×630px. Minimum: 200×200px. Facebook ignores images smaller than 200px. Use JPG or PNG for best compatibility.

og:url

Canonical URL

The canonical URL of your page. Facebook uses this to deduplicate shares — all shares of this URL will aggregate Likes and comments together in the social graph.

og:type

Content Type

Tells Facebook what kind of object this is. Common values: website, article, product, video.movie. Most pages should use website or article.

og:site_name

Site Name

Your brand or site name, shown in small text above the title. Helps establish brand recognition when your link is shared across Facebook. Keep it short and recognisable.

Image Requirements

Facebook OG Image Size Guide

Use the right image dimensions to ensure your link card always looks sharp on Facebook.

1200×630
✅ Recommended

Standard OG Image

The universal OG image size that works perfectly on Facebook, LinkedIn, Twitter/X, and most other platforms. Ratio: 1.91:1.

1080×1080
✅ Square Format

Square Image

Works well in Facebook feed and Instagram. Ratio 1:1. Facebook will crop this to 1.91:1 for standard link cards but keeps it square in carousel ads.

200×200
⚠️ Minimum

Minimum Size

Facebook will refuse to render images smaller than 200×200px. Images between 200px and 599px may render as small thumbnails rather than full-width cards.

How Does Facebook Read Open Graph Tags?

When a user shares a URL on Facebook — either by pasting it in a post, clicking Share, or using Facebook's Share Dialog — Facebook's crawler (called facebookexternalhit) visits the URL and reads the page's Open Graph meta tags from the HTML <head>. These tags tell Facebook what image, title, and description to display in the link card.

Our Facebook post preview tool simulates this process by rendering the tag values you input into a pixel-accurate mockup of a Facebook feed card, so you can see exactly how your content will appear before publishing.

The Correct OG Meta Tags Structure

<!-- Essential Facebook / Open Graph tags -->
<meta property="og:type" content="website" />
<meta property="og:url" content="https://your-page.com/" />
<meta property="og:title" content="Your Page Title — Brand" />
<meta property="og:description" content="A compelling description under 200 chars." />
<meta property="og:image" content="https://your-site.com/og-1200x630.jpg" />
<meta property="og:site_name" content="Your Brand Name" />
<meta property="og:locale" content="en_US" />

Facebook Link Card vs Facebook Ad Preview

It's important to understand that organic link cards (when someone shares a URL in a regular post) and Facebook Ads link card previews use the same OG tags but may render them slightly differently. In Ads Manager, you can set a custom headline and description that overrides OG tags. Our tool previews the organic link card — the experience someone sees when they share your URL in their own feed.

Why Is Facebook Showing Old Preview Data?

Facebook caches Open Graph data aggressively — sometimes for weeks. After updating your OG tags, you must use the Facebook Sharing Debugger (formerly Open Graph Debugger) to flush the cache for a specific URL. Enter your URL, click "Debug", then click "Scrape Again" to force Facebook to re-crawl and update the cached card data.

FAQ

Facebook Preview FAQs

The most common reasons: (1) og:image is missing or the URL returns an error, (2) the image is smaller than 200×200px, (3) the image URL requires authentication, (4) Facebook's cached version is stale — use the Sharing Debugger to re-scrape. Also ensure the og:image URL is absolute, not relative.
Facebook officially supports JPG, PNG, GIF (static), and as of 2021, WebP images. However, for maximum compatibility across all platforms (Twitter, LinkedIn, WhatsApp, etc.), JPEG or PNG with a .jpg/.png extension are still the safest choice.
Facebook's algorithm doesn't directly use og:description as a ranking signal, but a well-written description that encourages engagement (Likes, comments, shares) will indirectly improve your post's organic reach through Facebook's engagement-based feed algorithm.
Facebook caches Open Graph data for approximately 30 days by default. After updating your meta tags, use the Facebook Sharing Debugger (developers.facebook.com/tools/debug/) and click "Scrape Again" to force an immediate re-crawl of your URL.
