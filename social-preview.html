Open Graph Guide

How Social Platforms Read Your Meta Tags

Every time a URL is shared on a social platform, the platform's crawler visits your page and reads structured data from your HTML <head>. The primary protocol is Open Graph (OG), developed by Facebook and adopted by virtually every major social network.

Open Graph tags tell platforms what image, title, and description to use when building a link card preview. Without them, social networks guess — often showing a logo, a random image, or no image at all. With well-configured OG tags, you control the narrative completely.

<!-- Core Open Graph tags — add to every page's <head> --> <meta property="og:type" content="website" /> <meta property="og:url" content="https://your-page.com/slug" /> <meta property="og:title" content="Your Page Title — Brand" /> <meta property="og:description" content="Compelling description under 200 chars." /> <meta property="og:image" content="https://your-site.com/og-1200x630.jpg" /> <meta property="og:site_name" content="Your Brand Name" /> <!-- Twitter/X-specific tags --> <meta name="twitter:card" content="summary_large_image" /> <meta name="twitter:site" content="@yourtwitterhandle" />

Each platform interprets these tags slightly differently — different image ratios, different title lengths, different description truncation points. That's exactly why platform-specific preview tools are so valuable: one OG image doesn't look the same everywhere.
